# MihoyoBBSTools RS 使用说明

本说明面向直接使用程序的用户，适用于 Windows 与 Linux 发布包。

## 1. 软件用途

MihoyoBBSTools RS 可以按配置执行以下任务：

- 米哈游国内游戏签到。
- HoYoLAB 国际服游戏签到。
- 米游社社区签到。
- 可选的阅读、点赞、取消点赞和分享任务。
- 多账号依次运行并生成统一结果报告。
- 验证码平台接入。
- Telegram、Webhook、PushPlus 等多种通知。
- 账号代理及 Telegram 独立代理。
- 按日保存运行日志。

这是社区维护的非官方工具，与米哈游、HoYoverse 及其关联公司没有隶属、授权或合作关系。

## 2. 发布包内容

解压后目录如下：

```text
mihoyo-bbs-tools-<平台>-x86_64/
├── mihoyo-bbs-tools.exe     # Windows
├── mihoyo-bbs-tools         # Linux
├── 使用说明.md
└── config/
    └── config.example.yaml
```

Windows 使用 `mihoyo-bbs-tools.exe`，Linux 使用 `mihoyo-bbs-tools`。Linux 首次运行前可能需要授予执行权限：

```bash
chmod +x mihoyo-bbs-tools
```

命令中的相对路径以当前工作目录为基准。建议始终在程序所在目录运行。

## 3. 五分钟完成首次配置

### Windows

```powershell
cd 程序所在目录
.\mihoyo-bbs-tools.exe config add-account --name "主账号"
.\mihoyo-bbs-tools.exe validate-config
.\mihoyo-bbs-tools.exe run
```

### Linux

```bash
cd 程序所在目录
./mihoyo-bbs-tools config add-account --name "主账号"
./mihoyo-bbs-tools validate-config
./mihoyo-bbs-tools run
```

执行 `config add-account` 后粘贴完整 Cookie。程序会：

1. 校验 Cookie 格式并提取 UID 与 Stoken。
2. 查询米游社昵称。
3. 使用 `mys用户:<米游社昵称>` 作为配置中的账号名称。
4. 将 `--name` 内容保存为可选备注。
5. 在配置文件不存在时自动创建 `config/config.yaml`。

新账号默认只启用：

- 原神国内游戏签到。
- 米游社社区签到。

阅读、点赞、取消点赞、分享、HoYoLAB、云游戏和 Web 活动默认关闭。

## 4. Cookie 要求与安全

Cookie 必须完整，并至少能识别 UID、Stoken；V2 Stoken 通常还需要 MID。Cookie 不完整时，游戏签到可能可用，但社区任务、昵称查询或凭据刷新仍可能失败。

安全建议：

- 不要把 Cookie、Stoken、通知 Token、Webhook 或代理密码发到聊天、截图或公开仓库。
- 不要把 Cookie 直接写进命令参数，以免进入终端历史或进程列表。
- 推荐使用 `config add-account` 或 `config setup` 输入 Cookie。
- 配置和日志应只允许可信用户读取。
- Cookie 失效后使用设置菜单的“更新 Cookie”，程序会同时更新 Stoken 与昵称。

## 5. 配置文件

默认配置路径为：

```text
config/config.yaml
```

可以通过 `--config` 指定其他文件：

```powershell
.\mihoyo-bbs-tools.exe run --config D:\secure\mihoyo.yaml
```

配置有三种方式：

1. `config add-account`：首次添加账号最快捷。
2. `config setup`：推荐的完整数字菜单，可修改全部常用配置。
3. `config edit`：使用系统编辑器直接修改 YAML，适合熟悉 YAML 的高级用户。

也可以复制模板：

```powershell
Copy-Item config\config.example.yaml config\config.yaml
```

复制后必须替换示例 Cookie 或环境变量占位符。

每次修改后建议执行：

```powershell
.\mihoyo-bbs-tools.exe validate-config
```

## 6. 完整设置菜单

进入菜单：

```powershell
.\mihoyo-bbs-tools.exe config setup
```

一级菜单包括：

- 全局运行：请求超时、重试、随机延迟、日志级别、文件日志。
- 验证码：验证码平台端点。
- 账号：添加、删除、启用状态、备注、Cookie、设备、代理、任务、游戏。
- 通知：通用选项及通知渠道的添加、编辑、删除。
- 校验：立即检查当前配置。
- 高级 YAML 编辑：可选的完整文件编辑入口。

菜单输入规则：

- 留空通常表示保留当前值或取消当前操作。
- `-` 表示清空可选字段。
- `0` 表示返回上一级。
- 多选可以输入连续数字 `123`，也可以输入 `1,2,3`。
- Token、Cookie 与代理认证信息不会作为当前值回显。

## 7. 多账号

每个账号都有独立的：

- `name` 与 `remark`。
- 启用状态。
- Cookie 与 Stoken。
- 设备名称、型号、ID、FP。
- 网络代理。
- 任务开关。
- 游戏列表。

程序按配置顺序运行账号。账号名称一般为 `mys用户:<昵称>`；昵称重复时会追加 UID 尾号。备注只用于帮助用户区分账号。

添加账号：

```powershell
.\mihoyo-bbs-tools.exe config add-account --name "小号"
```

删除账号时使用配置中的账号名称：

```powershell
.\mihoyo-bbs-tools.exe config remove-account "mys用户:旅行者"
```

| `overseas_cloud_game` | 海外云游戏预留开关，当前未接入运行器 |
| `web_activity` | Web 活动预留开关，当前未接入运行器 |

游戏值：

| 值 | 游戏 |

## 9. 命令说明

查看总帮助：

```powershell
.\mihoyo-bbs-tools.exe --help
```

查看某个命令的详细参数：

```powershell
.\mihoyo-bbs-tools.exe run --help
```

常用命令：

| 命令 | 用途 |
|---|---|
| `version` | 显示版本、Git 提交与目标平台 |
| `validate-config` | 校验配置，不执行网络任务 |
| `run` | 执行签到、米游社任务并发送报告 |
| `checkin` | 只执行游戏签到 |
| `config setup` | 打开完整设置菜单 |
| `config add-account` | 添加账号 |
| `config remove-account` | 删除账号 |
| `config edit` | 高级 YAML 编辑 |
| `migrate-config` | 迁移旧版配置 |
| `print-example-config` | 输出完整默认模板 |

只运行指定任务：

```powershell
.\mihoyo-bbs-tools.exe run --task china-checkin
.\mihoyo-bbs-tools.exe run --task china-checkin,bbs
```

任务值为 `china-checkin`、`hoyolab-checkin`、`bbs`。命令行筛选只会缩小本次范围，不会重新启用 YAML 中已经关闭的任务。

只执行指定区域签到：

```powershell
.\mihoyo-bbs-tools.exe checkin --region china
.\mihoyo-bbs-tools.exe checkin --region hoyolab
.\mihoyo-bbs-tools.exe checkin --region all
```

## 10. 验证码平台

在 `config setup → 验证码` 中填写平台端点，或配置：

```yaml
captcha:
  endpoint: "https://example.com/captcha"
```

不使用验证码平台时设置为 `null`。端点可能包含访问凭据，应按敏感信息处理。

遇到验证码时，结果可能显示 `CaptchaRequired`。平台求解失败不会自动伪装成签到成功。

## 11. 代理

账号代理配置：

```yaml
proxy:
  url: "http://127.0.0.1:7890"
```

支持：

- `http://`
- `https://`
- `socks5://`
- `socks5h://`

省略协议时按 HTTP 代理处理。账号代理只作用于该账号的业务请求。

Telegram 可以配置独立代理：

```yaml
- type: telegram
  bot_token: "..."
  chat_id: "..."
  api_url: "https://api.telegram.org"
  proxy: "socks5h://127.0.0.1:1080"
```

Telegram 代理不会影响账号请求或其他通知渠道。

## 12. 通知

通知通用配置：

```yaml
notifications:
  enabled: true
  error_only: false
  block_keywords: []
  providers:
    - type: telegram
      bot_token: "..."
      chat_id: "..."
      api_url: "https://api.telegram.org"
      proxy: null
```

重要：`providers: []` 表示空列表。添加渠道时必须改成 `providers:` 并在下方缩进列表项，不能保留 `[]` 后再追加 `- type`。

通知渠道及主要字段：

| 类型 | 主要字段 |
|---|---|
| `telegram` | `bot_token`、`chat_id`、`api_url`、`proxy` |
| `webhook` | `url` |
| `pushplus` | `token`、`topic` |
| `ftqq` | `sendkey`、`api_url` |
| `pushme` | `token`、`api_url` |
| `cqhttp` | `url`、`qq` 或 `group` |
| `wecom` | `corp_id`、`agent_id`、`secret`、`to_user`、`api_url` |
| `wecomrobot` | `url`、`mobile` |
| `pushdeer` | `token`、`api_url` |
| `dingrobot` | `webhook`、`secret` |
| `feishubot` | `webhook` |
| `bark` | `token`、`api_url`、`icon` |
| `gotify` | `token`、`api_url`、`priority` |
| `ifttt` | `event`、`key`、`api_url` |
| `qmsg` | `key`、`api_url` |
| `discord` | `webhook` |
| `wxpusher` | `app_token`、`uids`、`topic_ids`、`api_url` |
| `serverchan3` | `sendkey`、`tags` |

CQHTTP 的 `qq` 与 `group` 必须二选一。WxPusher 的 `uids` 与 `topic_ids` 至少填写一项。Gotify 的 `api_url` 必填。

`error_only: true` 表示只在核心任务出现失败时推送。一个通知渠道失败不会阻止其他渠道，也不会覆盖任务本身的退出状态。

## 13. 日志

默认日志配置：

```yaml
runtime:
  log_level: info
  logging:
    enabled: true
    directory: logs
    file_prefix: mihoyo-bbs-tools
```

默认生成：

```text
logs/mihoyo-bbs-tools_2026-07-13.log
```

日期变化后会写入新的日期文件。旧配置中的 `file_prefix: mihoyo-bbs-tools.log` 仍兼容，也会生成新的下划线日期格式。

日志级别：`trace`、`debug`、`info`、`warn`、`error`。日志可能包含账号备注、角色 UID 尾号和执行结果，但不会主动输出 Cookie、Stoken、通知 Token 或代理认证信息。

## 14. 自动运行

建议先手动执行 `validate-config` 和 `run`，确认成功后再创建计划任务。

Windows 任务计划程序可设置：

```text
程序：D:\tools\mihoyo-bbs-tools.exe
参数：run --config D:\tools\config\config.yaml
起始于：D:\tools
```

Linux cron 示例：

```cron
15 8 * * * cd /opt/mihoyo-bbs-tools && ./mihoyo-bbs-tools run >> /dev/null 2>&1
```

避免同一配置被多个任务同时运行，以免重复请求或同时修改配置。

## 15. 旧配置迁移

位置参数方式：

```powershell
.\mihoyo-bbs-tools.exe migrate-config old.yaml new.yaml
```

兼容参数方式：

```powershell
.\mihoyo-bbs-tools.exe migrate-config --input old.yaml --output new.yaml
```

省略输出文件时，会在源文件同目录创建 `.migrated.yaml` 或 `.migrated.yml` 文件。程序不会覆盖已有目标文件。迁移结果可能包含真实凭据，应妥善保存。

## 16. 运行状态与退出码

常见状态：

- `Success`：任务成功。
- `AlreadyCompleted`：今日已经完成。
- `Skipped`：任务被关闭、没有绑定角色或无需执行。
- `AuthFailed`：Cookie 或凭据失效。
- `CaptchaRequired`：需要验证码。
- `NetworkFailed`：请求超时、连接失败或代理故障。
- `Failed`：其他业务失败。

主要退出码：

| 退出码 | 含义 |
|---|---|
| `0` | 成功、已完成或仅包含跳过项 |
| `1` | 普通任务失败 |
| `2` | 配置错误 |
| `3` | 认证失败 |
| `4` | 需要验证码 |
| `5` | 网络或代理失败 |

## 17. 常见问题

### YAML 报 `did not find expected key`

通常是缩进、冒号、引号或列表结构错误。重点检查报错行前几行。不要在 `providers: []` 后直接追加渠道。

### 环境变量未设置

配置中的 `${NAME}` 会在读取时展开。即使对应通知暂时关闭，只要占位符仍存在，也必须设置环境变量或改成实际值/null。

### 无法获取米游社昵称

确认 Cookie 完整且未过期，Cookie 中能识别 UID，并检查网络或代理。已有账号可在 `config setup → 账号 → 更新 Cookie` 中重试。

### Cookie 中缺少 Stoken

重新获取包含 Stoken 的完整 Cookie。只包含游戏签到字段的 Cookie 不足以添加完整账号。

### 显示“没有绑定角色”

该账号没有绑定对应游戏角色，或绑定区域不匹配。可以在设置菜单中关闭该游戏。

### Telegram 推送失败

检查 Bot Token、Chat ID、`api_url` 和 Telegram 独立代理。国内网络通常需要可访问 Telegram API 的代理。

### 日志没有生成

检查 `runtime.logging.enabled`、日志目录写入权限以及运行时工作目录。

## 18. 升级与备份

升级前建议备份：

```text
config/config.yaml
logs/
```

替换可执行文件后先运行：

```powershell
.\mihoyo-bbs-tools.exe version
.\mihoyo-bbs-tools.exe validate-config
```

不要直接用新模板覆盖真实配置。需要新增字段时，可对照发布包中的 `config/config.example.yaml`，或使用 `config setup` 修改。
